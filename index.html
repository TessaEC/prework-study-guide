HTML


  • The head element contains information about the webpage AKA Metadata.
    • Metadata> is data about our webpage that the browser uses to effectively serve our webpage to users
  • The opening tag and closing tag are represented with angle brackets (< header > =open < / header > =close)
    • In between these angle brackets will include the content and other nested elements
  • Example of some elements included in header:
    • < ! DOCTYPE html > tells web browser what doc type to expect on the page
    • < html lang = " en " > tells browser language is en for English
    • < title > defines title for webpage
    • < style > defines CSS style associates with website
    • < base > defines the base URL for a webpage
    • < link > connects an external resource to the HTML doc.
    • < meta > defines metadata
    • < script > embeds client-side scripts in the HTML doc.

  • The body element represents the visible content shown to the user AKA parent element
  • Nested elements within the body are called child elements
    • child elements are mostly semantic where the name of the element describes its content and the role it plays on the web page.
  • Examples of some semantic elements nested within the parent element:
    • < h1 > through < h6 > represents level of heading to appear on page
    • < p > paragraph
    • < ul > unordered list(bullet points) - if you use < ul > within list you can indent a list item
    • < li > listed items
    • < ol > ordered list(numbered/new indented line)
    • < b > bolds the word or words in between open and close
    • < u > underlines the word or words in between open and close
    • < img > contains info about an image that will show on the web page-can contain various attributes:
      • src = defines location of image to show on page
      • alt = provides description and functionality of the image (used for screen readers and other assistive technologies
        -can also improve engine rankings)
      • width = width of image
      • height = height of image
      • / > ends image information
    • < br / > shows line break on page
    • < br > breaks at inserted point of line - can wrap text to next line.
    • < section > defines a section in a doc. that contains thematic content
    • < a > anchor creates links to the same webpage or other pages and can contain other attributes:
      • href = points to the URL for the link, Example: < a hrf = " insert hyperlink " > insert text for hyperlink < / a >
  • Examples of non-semantic elements:
    • < div > to create a division in the page
    • < span > to create an inline division


CSS


  • Inline CSS - applies to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only
    • quick and permanent changes
    • less flexible than internal/external style sheets due to each line style must be edited individually.
  • Internal CSS - contains CSS rules for the webpage in the head section of the html file
    • Rules only apply to that page
    • Can configure CSS class and IDs-used to identify elements-to style multiple elements in the page code.
  • External CSS - Link from separate CSS file is placed in HTML head - Allows developers to keep all of their CSS rules in separate files
    • easier
    • most common
  • Separation of concerns - term in web dev. refers to separating code into diff. files based on use, the idea that ea. sections
    of code should have it's own responsibility.
  • Selector defines the element or attributes to which the rules, or declaration will apply.
  • Propertydictates which aspect of the selector will be changed visually.
    • Width declaration - makes element __ % of the available width (ex. width : 80% ;)
    • Margin declaration - assigns __px to the top and bottom of element (ex. margin : 40px auto ;)
      • Auto value = to the left and right
    • Border property - to border with width of __px + color of border (Ex. border : 5px solid gray ;)
    • Box Shadow - adds shadow effects around an elements frame. (Ex. box - shadow : 5px 10px blue ;)
  • Value is what the chosen CSS property will be changed to.
    • You can select colors by name OR by its hexadecimal code AKA hex #
  • Declaration contains 2 components = CSS property we want to apply and the value of the property.
  • Margin property indicates how much space we want around the outside of an element.
  • Padding property indicates how much space we want around the content inside an element.
  • CSS box rule - think of every HTML element as a box. Ea box consists of margins outside the element, and the element's
    borders, padding and content.
  • Do Not Repeat Yourself (DRY) code - a coding principal that reduces the number of lines of code that need to be written or maintained.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to an attribute with a selector, then the att. can be
    assigned to the element in html, enabling choice of which elements will get the CSS rule.
  • Selectors:
    • Asterisk symbol is a special wild card symbol - declares that ALL of the elements on the page will have the same style properties
    • Comma allows us to combine 2 or more selectors into a single CSS rule which allows the declarations to affect all selectors stated.
    • . card, the period designates this as a class selector. HTML elements assigned to card will get the property assignments from CSS
      • Example: applied declarations to CSS file-linked in html to each
        < section > as < section < class = " card " id = " git - section " >
  • To link an External CSS file to HTML:
    • we store metadata in the < head >
    • insert link to CSS file after < head > before < title > (Ex. < link rel = "stylesheet" href = " ./ assets / style . css" >)
      • link - creates connection
      • rel - specifies relationship b/w the current doc. and the linked doc. resource
      • href - specifies the location (URL) of the external resource-using a relative path which means the location is derived from the current location
        (index . html). To locate the style sheet from current location = assets / style . css(our file name).


Git


  • Git codes:
    • pwd: print working directory-shows what directory you are currently working in
    • ls: gives list of files and folders in your working directory(table of contents)
    • cd: change directory i.e. cd documents/ will change your WD to the documents folder
    • cd .: moves your WD back a level
    • mkdir bootcamp: Makes a new directory(folder), named bootcamp, in your WD
    • touch: creates new file, touch first-day.txt: creates new text(.txt) file named first-day
    • ls -al ~/.ssh: searches for an SSH key pair
    • git config --global user.name "insert a git username": sets a username for every repo on computer
    • git config --global user.name: to check your set user name
    • git config --global user.email "insert your email": sets an email address for every repo on computer
    • git config --global user.email: to check your set email address
    • git clone paste URL from GitHub: will add repo you created in GitHub
    • git remote -v: will show origin of repo fetch and push
    • code .: to open your WD in VS Code
    • git status: checks what branch you are currently on
    • git branch: will show you your branches
    • git checkout -b insert what you will call your new branch: creates new branch and will move you into that working branch
      (no spaces-use dash or underscore, no caps)
    • git branch: if typed again, it will show you all of your branches (title that is green is what branch you are currently on)
    • git checkout insert branch name: will move you into that branch(when you type git branch your new branch will now be green)
    • git branch --help: opens web browser page with info to help with branches
    • CL: will show any untracked folders/files that have not been added or synced to your new branch(will be red)
    • git add -A: will sync ALL untracked(red) files to your branch
    • git add insert specific file name: will only add that one specific untracked file
    • git commit -m"insert message i.e. add starter code": commits the files you had staged
    • git pull: to receive a branch's mods into the local environment
    • git push: you can create a new remote branch or add new changes to an existing one
  • Process to add new feature branch:
    • cd = change directory
    • cd bootcamp (main file)
    • cd prework - study - guide(file within bootcamp folder you want to branch off of)
    • git checkout main = branching off of main branch
    • git branch = checks what branch you are on - branch you want should be green
    • git branch -c feature/insert name you want new branch to be = will create your new feature branch
    • git checkout feature/new branch name = switches you to that branch
    • git branch = verifies what branch you are on - will be green
  • To delete a branch:
    • Move off of branch you are wanting to delete
    • git branch - - delete feature/name of branch
    • This will delete branch and any updates you have done since last merge to branch
  • Git flow:
    • Make sure you are on your feature branch
    • git add -A = stages all changed files to our feature branch
    • git commit - m " add message/note of what was changed "
    • git pull origin main = syncs our local branch with the remote branch in GitHub - always pull the base branch into your feature branch
      before pushing up any code
    • git push origin feature/name of branch = pushes our changes
    • refresh GitHub, your feature branch should show under main
    • Click pull requests tab at top
    • Click New Pull Request
    • Make sure the base is set to main and compare is set to your new branch
    • Click Create Pull Request
    • On the pull request form, give PR a title and short description of the changes
    • Click Create Pull Request
    • If working with a team on project you will tag others in your request and wait for approval before merging
    • If working on project alone, click Merge Pull Request and then delete branch
    • Close your issues list if it is completed after merge
